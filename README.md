# Task 3: SQL for Data Analysis

## 🎯 Objective
To extract, clean, and analyze customer data using SQL queries in MySQL Workbench, applying aggregation, joins, subqueries, and views for deeper insights.

## 🪜 Steps Performed
- Imported the dataset (`eecustomer.csv`) into MySQL Workbench.
- Explored table structure with `DESCRIBE` and previewed rows using `SELECT`.
- Ran queries with `WHERE`, `GROUP BY`, and `ORDER BY` for segmentation.
- Applied aggregate functions (`COUNT`, `AVG`, `SUM`) to calculate metrics.
- Used **subqueries** to filter high‑income, high‑spending customers.
- Created **views** for customer segmentation and analysis.
- Added **indexes** on key columns (`customer_id`, `gender`) to optimize queries.

## 📊 Queries Included
- **Query 1:** Total customers count  
- **Query 2:** Average income and spending score by gender  
- **Query 3:** Age group segmentation with average spending score  
- **Query 4:** Top 10 customers by spending score  
- **Query 5:** Missing values and duplicate checks  
- **Query 6:** Subquery for high‑income, high‑spending customers  
- **Query 7:** Created views for reusable analysis
